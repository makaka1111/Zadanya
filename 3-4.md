Техническое задание на создание Telegram-бота для кофейни

1. Введение
Настоящий документ формулирует требования к разработке Telegram-бота для локальной кофейни. Основное назначение бота — повышение посещаемости в периоды низкой загрузки заведения за счёт упрощённого процесса оформления заказов на кофе и сопутствующие товары. Решение будет функционировать как система онлайн-заказов с подключением платёжных сервисов и опциями доставки или самовывоза. Разработка ведётся в интересах владельцев кофейни (заказчик) и её клиентов (конечные пользователи). Проект нацелен на укрепление клиентской лояльности и оптимизацию потока посетителей.

2. Цели и задачи
Цели: Привлечение дополнительных клиентов в часы низкой активности (например, раннее утро или поздний вечер) благодаря удобству онлайн-заказа.

Рост среднего чека через возможность заказа нескольких позиций из ассортимента.

Создание уникального цифрового продукта, повышающего вовлечённость пользователей за счёт игровых и развлекательных элементов (дополнительные функции).

Задачи:

Разработать интерфейс для ознакомления с меню и выбора товаров.

Внедрить механизм формирования корзины с возможностью добавления нескольких товаров.

Интегрировать оплату через QR-код с последующим подтверждением транзакции.

Предоставить выбор способа получения заказа: самовывоз с указанием времени или доставка с вводом адреса.

(Дополнительно) Интегрировать модель искусственного интеллекта для неформального общения с пользователями (юмористический чат).

(Дополнительно) Реализовать систему NFT-стикеров в Telegram для закрепления уникальных позиций меню за пользователями.

3. Требования к функциональности
Бот должен реализовывать следующий функционал, организованный по модулям:

Каталог меню: Показ актуального меню кофейни (кофейные напитки, десерты, закуски) с описанием, ценами и картинками. Данные меню должны загружаться из БД или внешнего источника.

Работа с заказом: Пользователь может добавлять несколько позиций в корзину, изменять количество, удалять items. Состояние корзины сохраняется в течение сессии.

Оплата: Генерация QR-кода для перевода через внешний платёжный шлюз (например, ЮKassa или Tinkoff). После проведения платежа бот проверяет статус и информирует пользователя об успехе.

Способ получения:

Самовывоз: Выбор времени из доступных слотов.

Доставка: Ввод детального описания точки доставки (адрес, комментарии). Бот проверяет доступность зоны доставки на основе геоданных.

Подтверждение заказа: После успешной оплаты и выбора способа — отправка итогового чека клиенту и уведомление персоналу кофейни.

(Доп.) Чат с ИИ: Интеграция с крупной языковой моделью (LLM, например, Grok или аналог) для генерации шутливых ответов на реплики пользователей, не связанные с заказом. Реализация через API с ограничением запросов для контроля расходов.

(Доп.) NFT-стикеры: Создание уникальных стикеров на основе NFT (с использованием блокчейна, например, TON или Ethereum). Каждый стикер ассоциируется с эксклюзивной позицией меню (например, "фирменный раф"). Пользователь получает стикер после заказа соответствующего товара, и он привязывается к его аккаунту в Telegram. Реализация: минт NFT через смарт-контракты, интеграция с Telegram Sticker API.

4. Требования к интерфейсу

UI в Telegram: Использование стандартных элементов Telegram Bot API (кнопки, инлайн-клавиатуры, сообщения с медиа). Меню отображается в виде списка или карусели кнопок.

Навигация: Простая и понятная цепочка: главное меню → категории товаров → корзина → оплата → выбор получения → подтверждение.

Визуал: Изображения товаров (превью), эмодзи для обозначения категорий. Тексты сообщений лаконичные, дружелюбные.

Адаптивность: Корректная работа на мобильных устройствах (iOS/Android). Кастомизированный графический интерфейс не требуется, только текстово-кнопочный.

(Доп.) Для доступа к ИИ — отдельная кнопка "Пошутить". Для NFT — отображение полученного стикера в чате со ссылкой для просмотра в кошельке.

5. Нефункциональные требования

Производительность: Время отклика бота на действие пользователя — не более 2 секунд. Поддержка до 100 одновременных сессий.

Надёжность: Автоматический перезапуск при сбоях Telegram API. Ведение логов ошибок.

Безопасность: Шифрование данных заказов при хранении. Соблюдение требований GDPR/ФЗ-152 (применимо). Платёжные данные в боте не хранятся.

Совместимость: Работа с Telegram Bot API версии 6.0+. Поддержка русского и английского языков интерфейса.

Документация: Инструкция для администратора по управлению меню. Встроенная справка для пользователей (команда /help).

Масштабируемость: Возможность добавления новых товаров в меню без остановки бота.

(Доп.) Для ИИ-чата — фильтрация недопустимых запросов. Для NFT — гарантия уникальности токенов через верификацию в блокчейне.

6. Интеграции и среда выполнения
Интеграции:

Telegram Bot API (основной функционал).

Платёжный шлюз (ЮKassa или аналогичный) для генерации QR-кодов.

База данных (SQLite/PostgreSQL) для хранения данных меню и заказов.

API картографических сервисов (Яндекс.Карты или Google Maps) для валидации адресов доставки.

(Доп.) API языковой модели (например, xAI Grok API) для чата.

(Доп.) Блокчейн-платформа (TON/Ethereum) для NFT, интеграция с Telegram Wallet.
Среда выполнения:

Развёртывание на сервере (VPS, AWS, Heroku) на стеке Python/Node.js.

Доступ для пользователей через приложение Telegram на мобильных устройствах.

Тестирование в sandbox-режиме Telegram.

7. Ограничения и допущения
Ограничения:

Доставка осуществляется только в границах города (радиус до 10 км от кофейни).

Оплата принимается исключительно через QR-код, без встроенной оплаты картой в интерфейсе бота.

В меню не используются видео- или аудиофайлы.

(Доп.) ИИ-чат ограничен 5 сообщениями за сеанс для экономии ресурсов.

(Доп.) NFT-стикеры выпускаются только для премиальных позиций, не для всего ассортимента.
Допущения:

Заказчик предоставит актуальные данные меню и изображения товаров.

Техническая поддержка и обновления бота после сдачи проекта лежат на заказчике.

Пользователи используют Telegram версии не ниже 8.0.

Бот не несёт ответственности за задержки, связанные с работой кофейни (доставка/приготовление).

8. Критерии приёмки

Функциональное тестирование: Все основные сценарии (просмотр меню, работа с корзиной, оплата, выбор доставки) выполняются без ошибок. Провести тестовые 10 заказов.

Интерфейс: Корректное отображение на Android и iOS. Отсутствие нарушений в логике навигации.

Нефункциональные показатели: Время отклика ≤2 сек, доступность 99% в течение тестовой недели.

(Доп.) Бонусные функции: ИИ-чат генерирует осмысленные и уместные ответы (проверка вручную на 5 запросах). NFT-стикеры успешно создаются и отображаются у пользователя.

Общие: Полное соответствие требованиям данного ТЗ. Подписание акта выполненных работ после демонстрации работоспособной системы.
Проект считается завершённым после успешного прохождения всех тестов и получения подтверждения от заказчика.

9. Приложения

Схемы: Диаграмма потока выполнения заказа (прилагается файл order_flow.png, если имеется).

Макеты интерфейса: Примеры инлайн-клавиатур (описаны текстом, графические макеты — по отдельному запросу).

Сроки и бюджет:

Разработка: 4 недели.

Тестирование и отладка: 1 неделя.

Бюджет: Формируется на основе данного ТЗ (предварительная оценка — 500 000 руб., включая реализацию дополнительных функций).

Дополнения: Возможные изменения в требованиях — только по согласованию сторон, с соответствующим пересмотром сроков и бюджета.

Комментарии к бонусным реализациям:

ИИ-чат: Использование предобученной модели (Grok) с дообучением на юмористических диалогах. Ограничение запросов по токенам через API.

NFT: Развёртывание смарт-контракта на блокчейне TON для чеканки токенов. Процесс: после оплаты бот инициирует минт NFT и отправляет пользователю ссылку на стикер-пак.
